# Monetization Spec

## Purpose

Define a fair monetization strategy for rakugaking.

## Core Principle

The game must never become pay-to-win.

Players who pay should gain more ways to play and customize, not stronger fighters.

## Monetization Goals

- Support long-term development.
- Keep the full battle system available to all players.
- Reward creativity instead of spending.

## Candidate Paid Content

### Character Slots
- Additional save slots.

### Body Types
- Dog
- Bird
- Lizard
- Giant
- Monster

Each body type changes how players draw but should remain balanced.

### Drawing Tools
- Extra brushes
- Decorative stamps
- Premium colors

### Visual Content
- Arena themes
- Battle effects
- Victory animations
- UI themes

## Online

Online play itself should not require payment.

Optional cosmetic purchases may be shown online, but matchmaking should not depend on purchases.

## Not Allowed

- Paid attack power
- Paid HP
- Paid command strength
- Paid exclusive competitive advantages

## Future Considerations

- Season pass
- Creator support packs
- Collaboration cosmetics

## Design Rule

If an item changes battle performance, it should be obtainable through gameplay as well as purchases, or be removed from monetization entirely.
