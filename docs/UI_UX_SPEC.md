# UI / UX Spec

## UX Principle

rakugaking must feel easy, playful, and funny.

The player should not feel like they are using a professional 3D tool or writing code.

## Main Flow

1. Create character
2. Draw parts
3. Assign simple commands
4. Start battle
5. Watch result
6. Save or share character

## MVP Screens

## Home

Purpose:

- Start creating a character
- Start battle
- View saved characters

Main actions:

- Create Character
- Battle
- Character List

## Character Drawing Screen

Purpose:

- Draw each body part.

UX requirements:

- Show current part clearly.
- Let player redraw a part easily.
- Do not require precision.
- Keep controls simple for smartphone.

## Assembly Preview Screen

Purpose:

- Show how drawn parts become a character.

UX requirements:

- Player can preview the full character.
- Player can see if parts are connected.
- Player can proceed even if the character looks strange.

## Command Setup Screen

Purpose:

- Assign simple commands to parts.

UX requirements:

- Use buttons/cards, not text coding.
- Explain commands with simple labels and icons.
- Avoid advanced programming terms.

Example:

- Arm: Punch
- Arm: Guard
- Leg: Kick
- Leg: Jump

## Battle Screen

Purpose:

- Watch automatic battle.

UX requirements:

- Camera should make the whole fight readable.
- Character actions should be funny and understandable.
- Player should know who is winning.

Minimum HUD:

- Character A name
- Character B name
- HP or status
- Timer

## Result Screen

Purpose:

- Show battle outcome.

UX requirements:

- Show winner/draw.
- Offer rematch.
- Offer edit character.
- Offer save/share if available.

## Online / Sharing UX Direction

Future screens may include:

- shared character gallery
- friend battle
- online matchmaking
- character code/import

These are important product goals but should be designed after the core loop is stable.

## Monetization UX Direction

Monetization should be presented as optional fun expansion.

Avoid:

- pay-to-win wording
- blocking the first core battle
- making children accidentally buy unclear items

## Accessibility

- Use large touch targets.
- Avoid tiny text.
- Keep instructions short.
- Support simple icons.
